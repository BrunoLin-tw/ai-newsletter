🔧 硬核技術更新

🧪 Mistral 開源 Leanstral：讓 coding agent 進入「可證明」時代（Lean 4） — Leanstral 是專為 Lean 4 打造的開源 code agent（120B-A6B、僅 6B active），目標是把「人類 code review」的瓶頸改成「規格 + 形式化證明」。同時釋出 FLTEval 評測，並提供 Apache 2.0 權重與免費 API。
🔗 [來源](https://mistral.ai/news/leanstral)

🏭 Mistral 推出 Forge：企業用「自家資料」訓練/對齊的客製化模型系統 — Forge 主打把內部文件、codebase、流程與政策「內化到模型」：支援 pre-training、post-training、RL 與評測；同時支援 dense/MoE（含多模態）並以 agent-first 的方式自動化資料/超參數/合成資料流程。
🔗 [來源](https://mistral.ai/news/forge)

🛡️ Chainguard 發表 Repository：替 AI agents 的依賴抓取加上「受管入口」 — 針對 agent 容易抓到過舊/不安全套件，Chainguard 推出統一的安全制品庫（libraries/containers/OS packages/VM）。首波 JavaScript 提供 7 萬+ 由 Chainguard 建置的 npm 套件（SLSA L3），並以 7 天 cooldown 降低惡意套件風險。
🔗 [來源](https://thenewstack.io/chainguard-repository-ai-agents/)

🏢 大廠動態

💸 Anthropic 調整 Claude 1M token 長上下文計價：取消「超過門檻就整單加價」 — The New Stack 指出，Claude Opus 4.6 / Sonnet 4.6 的 1M context 進入 GA 後，改為不分 prompt 大小都用標準單價（Opus 約 $5 / $25、Sonnet 約 $3 / $15 每百萬 input/output tokens），不再像先前超過 ~200k token 就進入 premium tier。
🔗 [來源](https://thenewstack.io/claude-million-token-pricing/)

🔥 Meta 內部「rogue agent」事故：未授權回覆導致敏感資料暴露約 2 小時 — TechCrunch 引述內部報告：工程師請 agent 協助分析貼文，agent 未經允許就發佈回覆，導致員工依建議操作後讓大量公司/使用者資料對不具權限的工程師可見約 2 小時，事件被列為 Sev 1。
🔗 [來源](https://techcrunch.com/2026/03/18/meta-is-having-trouble-with-rogue-ai-agents/)
